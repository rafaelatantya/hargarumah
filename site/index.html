Eight sites. No consolidated view.

Indonesian property data lives in a dozen separate platforms. Each has its own URL pattern, DOM structure, and rate limit. Manual collection does not scale past a single neighborhood.

Architecture

Five stages from coordinates to structured data.

01

Orchestrator

Coordinates the 8 scrapers, validates through Pydantic, deduplicates at storage, exports to JSON, CSV, and XLSX.

async def run_scrapers(
    keyword: str,
    site_filter: str = "all",
    min_listings: int = 10,
) -> None:
    areas = await geo.resolve(keyword)
    async with BrowserManager() as browser:
        for scraper in get_scrapers(site_filter):
            listings = await scraper.scrape(
                areas, browser, min_listings
            )
            await storage.upsert_many(listings)
    await exporter.export_all(
        formats=["json", "csv", "xlsx"]
    )
02

Geo resolver

Coordinate to area name, recursive address parsing from street to kecamatan. Falls back to broader levels if the specific search yields too few listings.

03

Browser

nodriver starts a real Chrome via CDP. No webdriver flag, no chromedriver binary, no Selenium properties. Headed mode preferred for stealth.

04

Eight scrapers

One module per site, all inheriting BaseScraper. tab.evaluate JSON.stringify, json.loads back in Python. Strict location filter applied to drop false positives.

Storage, export

Async SQLite for persistence and dedup. JSON for downstream pipelines, CSV for spreadsheets, XLSX for humans.

Stealth by design

CDP, not WebDriver

nodriver talks to Chrome via DevTools Protocol. No navigator.webdriver flag, no chromedriver binary. Real browser fingerprint, most JS-based bot checks pass.

Random 2 to 5 second delays

Between navigations, between scraping sessions, between different sites. Never the same interval twice in a row.

Optional proxy rotation

Free proxy lists pulled on demand, validated before use, rotated per session. Falls back to direct connection if proxies fail.

Twenty-two fields, validated

Every listing passes through a Pydantic model before storage. Price formats, area units, and address strings normalize to one canonical shape.

src / models / property.py 
class PropertyListing(BaseModel):
    # Required
    id: str
    source: str
    title: str
    price_idr: int = Field(..., ge=0)
    url: str
    scraped_at: datetime

    # Location
    address: str | None = None
    city: str | None = None
    district: str | None = None
    latitude: float | None = None
    longitude: float | None = None

    # Specs
    bedrooms: int | None = None
    bathrooms: int | None = None
    land_area_m2: float | None = None
    building_area_m2: float | None = None

    # Indonesian price parser
    "Rp 850 Jt"  ->  850_000_000
    "Rp 1,2 M"   ->  1_200_000_000
    "2 Miliar"   ->  2_000_000_000
